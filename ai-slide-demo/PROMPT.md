# Prompt — AI-Native Slide Systems

Reconstructed prompt for the two decks in this folder — the originals weren't
preserved, so these prompts are written to replicate the samples.

---

## Deck 1 — `index.html`

Create a keyboard-driven HTML slide deck titled "AI-native slides are
systems, not files." arguing that AI should generate reusable slide systems
rather than one-off decks.

Slides cover: blank slides drain the work → make brand consistency reusable
("Claude Designs") → HTML decks are presentation-as-code → reusable formats
beat one-off slides → how to generate a better first draft → HTML makes
slides more than static images → two useful paths forward.

One self-contained HTML file: arrow-key + click navigation, slide counter,
progress indication, smooth slide transitions, strong typographic hierarchy.
No external dependencies.

## Deck 2 — `mcp-is-dead.html`

Same deck system, different content: an opinion deck titled
'"MCP is dead." — Is it though?' that steelmans and then rebuts the claim,
walking through what MCP actually standardizes, where the criticism is right,
and what would have to exist for it to die. Same technical constraints.
